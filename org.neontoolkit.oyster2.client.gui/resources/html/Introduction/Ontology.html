This class represents metadata about ontologies themselves.
NameRangeCardinalityDescription
URI xsd:string1:1The URI of the ontology which is described by this metadata. It serves as a logical identifer and is not necessarily the physical location
name xsd:string 1:n The name by which an ontology is formally known
acronym xsd:string 1:1 A short name by which an ontology is formally known
description xsd:string 1:1 Free text description of an ontology
documentation xsd:string 0:1 URL for further documentation
notes xsd:string 0:1 Describes additional information about the ontology that is not included somewhere else (e.g. information that you do not want to include in the documentation)
keywords xsd:string 0:n List of keywords related to an ontology
keyclasses xsd:string 0:n Indicates what the central/best represented classes of the ontology are
status xsd:string 0:1 It specifies the tracking information for the contents of the ontology
creationDate xsd:date 1:1 Date when the ontology was initially created.
modificationDate xsd:date 0:1 Date of the last modification made to the ontology
hasContributor omv:Party 0:n Contributors to the creation of the ontology
hasCreator omv:Party 1:n Main responsible for the creation of the ontology
usedOntologyEngineeringTool omv:OntologyEngineeringTool 0:n Information about tool used to create the ontology
usedOntologyEngineeringMethodology omv:OntologyEngineeringMethodology 0:n Information about the method model used to create the ontology
usedKnowledgeRepresentationParadigm omv:KnowledgeRepresentationParadigm 0:n Information about the paradigm model used to create the ontology
endorsedBy omv:Party 0:n Indicates the parties (i.e. organisations, people) that have expressed support or approval to the this ontology.
hasDomain 0:n Specifies the domain topic of an ontology
isOfType omv:OntologyType 1:1 The nature of the content of the ontology
naturalLanguage xsd:string 0:n The language of the content of the ontology, i.e. English, German, etc.
designedforOntologyTask omv:OntologyTask 0:n Declares for which purpose the ontology was originally designed
hasFormalityLevel omv:FormalityLevel 0:1 Level of formality of the ontology
knownUsage xsd:string 0:n The applications where the ontology is being used
hasOntologyLanguage omv:OntologyLanguage 1:1 Specifies the ontology language
hasOntologySyntax omv:OntologySyntax 1:1 It specifies the presentation syntax for the ontology language
isConsistentAccordingToReasoner xsd:boolean 0:1 Indicates whether a reasoner has classified the ontology correctly
expressiveness xsd:string 0:1 Indicates the expressiveness of the language used by the ontology (that could be expressed in terms of the underlying DL, e.g. ALCN(D), SHIF(D), etc.)
resourceLocator xsd:string 1:n The location where the ontology can be found. It should be accessible via a URL
version xsd:string 1:1 Specifies the version information of the ontology. It is highly recommended the use of a well defined numbering schema for the version information (e.g. X.Y.Z where X is a major release, Y is minor release and Z is a revision number)
hasLicense omv:LicenseModel 0:1 Underlying license model
useImports omv:Ontology 0:n References another ontology metadata instance that describes an ontology containing definitions, whose meaning is considered to be part of the meaning of the ontology described by this ontology metadata instance. Each reference consists of a URI
hasPriorVersion omv:Ontology 0:1 Contains a reference to another ontology metadata instance
isBackwardCompatibleWith omv:Ontology 0:n This property identifies the ontology metadata instance which describes an ontology that is a compatible prior version of the ontology described by this ontology metadata instance
isIncompatibleWith omv:Ontology 0:n This property indicates that the described ontology is a later version of the ontology described by the metadata specified, but is not backward compatible with it. It can be used to explicitly state that ontology cannot upgrade to use the new version without checking whether changes are required.
containsTBox xsd:boolean 0:1 Indicates if the ontology contains assertions about concepts (e.g. subsumption, equivalence)
containsRBox xsd:boolean 0:1 Indicates if the ontology contains assertions about roles (properties) and role hierarchies (e.g. subsumption, equivalence, transitivity)
containsABox xsd:boolean 0:1 Indicates if the ontology contains role assertions between individuals and membership assertions (e.g. concept instantiation , role instantiation)
numberOfClasses xsd:unsignedLong 1:1 number of classes in the ontology
numberOfProperties xsd:unsignedLong 1:1 number of properties in the ontology
numberOfIndividuals xsd:unsignedLong 1:1 number of individuals in the ontology
numberOfAxioms xsd:unsignedLong 1:1 number of axioms in the ontology
